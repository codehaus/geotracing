GPSDoor

GPSDoor is an application that provides access to a GPS locally attached to your machine. It's the door to your GPS. If you follow the steps below you can be up and running in a minute. The current version is Mac OSX-only.

The Steps (MAC OSX)

Prerequisites

  • Mac OS X (10.3 and higher)
  • Bluetooth GPS module (USB GPS may use USB to serial)
  • switch on your GPS and wait for a fix

Download GPSDoor

  • download gpsdoor-1.0.0.zip

Install GPSDoor

Unpack the zip-file and place the application anywhere you like, for example in Applications folder.

Pair your GPS with your Mac

  • in Bluetooth menu "Set up Bluetooth Device"
  • setup as "Any" and follow wizard-steps
  • if asked for a passkey: most devices have "0000" otherwise consult your GPS documentation

Make your GPS available as a device

This step is only required for OS X 10.3.x (Panther ) and lower. OS 10.4.x (Tiger) already has created a serial device during the pairing step ("Set up Bluetooth Device") above. What happens is that your Bluetooth GPS becomes a serial device like /dev/tty.HoluxGPSlim236-SPP1-Slave.

  • open Applications/Utility/Bluetooth Serial Utility
  • do New...
  • give a sensible name, e.g. "HoluxGPS"
  • select device and select the above paired Bluetooth GPS
  • set Port Direction to "Outgoing" (yes really!)
  • leave options blanc (auth, encryption, network preferences)
  • set Port type to RS-232

Run and check GPSDoor

  • Start GPSDoor by clicking icon.
  • the GPSdoor window will on first use show that your GPS is not found
  • your browser (usually Safari) will start on the page http://localhost:7305
  • Select your GPS from the dropdown list on the right (something like "/dev/tty.HoluxGPSlim236-SPP1-Slave")
  • the GPSDoor window should show ++++++ (each + means a valid GPS-sample)
  • a new map will show you on your current location
  • you can now start a WebTracer-session (see next)

Start a WebTracing Session

start a session. If you don't have an account create an account by registering

Notes

  • GPSDoor saves selected config in /Users/[user]/gpsdoor.ini
  • sockets are not always closed properly, just wait after restart. GPSDoor will retry.
  • Do not use "Cancel" button to stop GPSDoor but use "Quit" in dock or "Shutdown" in http://localhost:7305
  • Safari is the default browser but does not work well in all cases (at least on MAC OS X 10.3 Panther). Please use FireFox or Mozilla.

Technical

GPSDoor is a Python script that reads samples from your GPS using file-based (Unix-style) device access. At the same time GPSDoor runs a small webserver (on localhost port 7305) that makes GPS-data and status/management available as web-services. One of the webservices (http://localhost:7305/js) provides a JavaScript file that sets coordinates in your WebTracer web-page. The WebTracer fetches these coordinates periodically using GPS.js, sending them to the GeoTracing server within your session.
